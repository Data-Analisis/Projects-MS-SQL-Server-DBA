SRV-проект БД для администрирования СУБД (DBA)

MailBiblio - библиотека для отправки писем

ExcelBiblio - библиотека для работы с Excel любой версии (с 1997) (необходимо, где запускается приложение, наличие установленного Excel Office)

RunExecuteInAllDB - запуск запроса на все БД всех указанных экземпляров скуля:
1) Servers - перечислите названия экземпляров скуля через ;
2) FileQuery - файл, где лежит запрос
3) FileLog - файл для лога

**********************************************************************

BackupRefreshDB - решение для работы с БД:

DBFileFunctions - билиотека функций для работы с БД

LogShipping - настройка доставки журналов транзакций:
1) SourceServer - название экземпляра скуля от куда берем БД
2) TargetServer - название экземпляра скуля куда доставляем БД
3) SourceFolderShared - общая папка доступная (\\путь) на первом экземпляре
4) TargetFolderShared - общая папка доступная (\\путь) на атором экземпляре
5) IsSourceServer- делать ли резервную копию БД с SourceServer
6) SourceFile - список полных имен файлов резервных копий БД, если IsSourceServer=false
7) ListDB - список пар БД_источник;БД_приемник, задающий что переносить и в какую БД
8) FileLog - файл для лога
9) IsDeleteSourceBackup - удалять ли резервную копию на источнике
10) MoveTo - путь куда перемещать файлы (локальный путь)
11) IsDeleteTargetBackup - удалять ли резервную копию на получателе
12) IsMail - отправлть ли результат на почту
13) MailAddressFrom - почта от кого
14) MailAddressTo - почта кому
15) MailServer - сервер почтовый
16) IsCopySource - признак того, что нужно копировать БД с источника
17) IsRestoreTarget - признак того, что необходимо восстанавливать БД на получателе
18) IsLogShipping - настраивать ли доставку журналов транзакций
19) FileRetentionPeriodMin - период хранения файлов в минутах
20) ThresholdMin - минимальный порог в минутах (по преодолении его должна генерироваться сообщение о проблеме)
21) HistoryRetentionPeriodMin - сколько хранить историю по времени в минутах
22) MonitorServer - экземпляр скуля для монитора
23) Login - логин для настройки доставки журналов транзакций (при IsSQLLogin=true)
24) Password - пароль для настройки доставки журналов транзакций (при IsSQLLogin=true)
25) RunMin - как часто выполнять по времени в минутах
26) IsSQLLogin - логин и пароль скульный

BackupRefreshDB - настройка зеркалирования, создания резервной копии, копирования на другой сервер, восстановления БД (одна супница):
1) SourceServer - сервер-источник скульный
2) TargetServer - сервер-получатель скульный
3) SourceFolderShared - доступная для общего использования папка (\\путь) на источнике
4) TargetFolderShared - доступная для общего использования папка (\\путь) на получателе
5) IsSourceServer - делать ли резервную копию БД на источнике
6) SourceFile - если не нужно делать резервную копию БД на источнике, то указывется список полных имен файлов-резервных копий БД
7) TargetServerMirror - сервер скульный для зеркала
8) TargetFolderSharedMirror - доступная для общего использования папка (\\путь) на зеркале
9) IsMirror - настраивать ли зеркалирование
10) ListDB - список пар БД_источник;БД_приемник, задающий что переносить и в какую БД
11) FileLog - файл для лога
12) IsDeleteSourceBackup - удалять ли резервную копию на источнике
13) MoveTo - путь локальный куда перенести файлы БД на получателе (локальный путь)
14) MoveToMirror - путь локальный куда перенести файлы БД на зеркале (локальный путь)
15) IsDeleteTargetBackup - удалять ли резервную копию на получателе
16) IsMail - отправлть ли результат на почту
17) MailAddressFrom - почта от кого
18) MailAddressTo - почта кому
19) MailServer - сервер почтовый
20) SourceStringTCP_Mirror - конечная точка для зеркала
21) TargeStringTCP_Mirror - конечная точка для источника
22) IsMirrorAsync - признак асинхронности режима зеркалирования
23) IsCopySource - признак того, что нужно копировать БД с источника
24) IsRestoreTarget - признак того, что необходимо восстанавливать БД на получателе
